# プロンプトテンプレート

タスクプロンプトを作成する際に使用するテンプレート集です。

## テンプレート一覧

| テンプレート | 用途 |
|------------|------|
| [feature-task.md](./feature-task.md) | 新機能の追加（大規模、詳細な設計が必要） |
| [refactor-task.md](./refactor-task.md) | リファクタリング（デザインパターンの導入など） |
| [simple-task.md](./simple-task.md) | シンプルなタスク（小規模な変更） |
| [bugfix-task.md](./bugfix-task.md) | バグ修正 |

## 使い方

1. タスクの種類に応じて適切なテンプレートを選択
2. テンプレートをコピーして `tasks/` ディレクトリに新しいファイルを作成
3. `[プレースホルダー]` を実際の内容に置き換える
4. 不要なセクションは削除してOK

## テンプレート選択の目安

### feature-task.md を使う場合
- 新しいモジュールや機能を追加する
- 外部APIやライブラリとの統合が必要
- 複数のPhaseに分かれる大規模な実装
- アーキテクチャの設計が必要

### refactor-task.md を使う場合
- 既存のコードを改善する
- デザインパターンを導入する
- コードの構造を変更する（機能は変えない）
- テスタビリティや拡張性を向上させる

### simple-task.md を使う場合
- 小規模な変更や追加
- 設定の変更
- ドキュメントの更新
- 依存パッケージの更新

### bugfix-task.md を使う場合
- バグの修正
- 不具合の対応
- エッジケースの対応

## 完了したタスク

完了したタスクは `tasks/completed/` ディレクトリに移動してください。
